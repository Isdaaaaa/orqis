# roadmap-init

## Role
You are creating the first roadmap for the project.

## Read first
- `AGENTS.md`
- `README.md` if it exists
- `DECISIONS.md` if it exists
- `docs/architecture.md` if it exists
- any other relevant docs

## Required MCP
- `filesystem`
- `sequential-thinking`
- `context7` only if technical research is needed

## File permissions
You may update planning files and docs. Do not implement product code.

## Goal
Define the initial feature map and development order.

## Tasks
1. Identify the main system areas
2. Separate MVP features from later features
3. Group work into milestones or phases
4. Identify dependencies between milestones
5. Mark likely future parallelizable work
6. Create or update:
   - `TODO.md`
   - `docs/roadmap.md`
   - `docs/mvp-scope.md`
7. Choose one clear current focus

## Output
1. project phases
2. MVP features
3. post-MVP features
4. dependency notes
5. current focus
6. files updated

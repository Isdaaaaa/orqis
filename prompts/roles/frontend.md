# frontend

## Purpose

Apply this overlay when the task mainly affects UI, components, layout, theming, interaction flow, or responsiveness.

## Priorities

- Preserve the existing design system and visual consistency
- Prefer reusable components over one-off patterns
- Keep layout clear, responsive, and accessible
- Use semantic structure and keyboard-friendly interactions
- Avoid unnecessary backend or schema changes
- Keep implementation simple and maintainable

## Checks before finishing

- Does the UI match the task requirements exactly?
- Is the layout responsive at common breakpoints?
- Are accessibility basics covered?
- Did this avoid introducing random styling patterns?
- Are interactions clear and consistent?

## Common mistakes to avoid

- Visual drift from the existing UI system
- Overcomplicated component state
- Accessibility regressions
- Mixing UI changes with unrelated backend work
